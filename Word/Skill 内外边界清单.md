# Skill 内外边界清单

## 1. 文档目标

本文档用于明确未成年人识别项目中哪些能力属于：

- 正式 Skill 本体
- 外部运行时系统
- 研发工具链
- 网站体验层

目标是避免后续开发中继续出现以下混淆：

- 什么应该写进 `SKILL.md`
- 什么应该做成 `scripts/`
- 什么应该留在数据库或运行时系统
- 什么只是研发优化工具，不应进入交付物

---

## 2. 一句话边界原则

- **Skill** 负责“如何分析”
- **运行时系统** 负责“如何准备上下文并调 Skill”
- **研发工具链** 负责“如何优化 Skill”
- **网站** 负责“如何展示 Skill 的结果”

---

## 3. 总体边界表

| 能力/组件 | 归属 | 是否进入最终 Skill 包 | 说明 |
| --- | --- | --- | --- |
| 触发 description | Skill | 是 | 用于何时激活 Skill |
| 主判定规则 | Skill | 是 | 未成年人识别核心逻辑 |
| ICBO 提取规则 | Skill | 是 | 输出结构的一部分 |
| 用户建模逻辑 | Skill | 是 | 基于输入会话或历史做画像推断 |
| 多会话趋势分析 | Skill | 是 | 基于传入历史会话输出轨迹 |
| 时间特征脚本 | Skill | 是 | 作为 Skill 包内可选规范化脚本存在 |
| 调用时间脚本 | Skill | 是 | 作为 Skill 内部工作流的一部分按条件调度 |
| 内置 RAG 脚本 | Skill | 是 | 可选增强能力 |
| retrieval assets（导出后的检索资产） | Skill | 是 | 可从训练集导出，作为内置 RAG 的运行资源 |
| 外置 RAG 服务接入 | 运行时系统 | 否 | 宿主系统可选提供外部检索结果，不进 Skill 本体 |
| 长期画像数据库 | 运行时系统 | 否 | 持久化能力 |
| 历史摘要构造 | 运行时系统 | 否 | 为 Skill 提供 `prior_profile` |
| 结果归档 | 运行时系统 | 否 | 保存每次分析结果 |
| 体验网站页面 | 网站系统 | 否 | 展示层能力 |
| evaluator | 研发工具链 | 否 | 只用于评测 |
| optimizer | 研发工具链 | 否 | 只用于迭代 |
| pipeline | 研发工具链 | 否 | 只用于研发流程 |
| benchmark 数据集 | 研发工具链 | 否 | 评测资产 |

## 4. Skill 内部边界

### 4.1 必须属于 Skill 的内容

#### 触发定义

包括：

- `name`
- `description`
- 可选 compatibility 说明

原因：

- 这是 Skill 的发现与激活基础

#### 主判定逻辑

包括：

- 年龄线索识别
- 学段线索识别
- 成年误导线索识别
- 证据冲突处理
- 置信度生成规则

原因：

- 这是 Skill 的核心价值

#### ICBO 抽取规则

包括：

- intention
- cognition
- behavior_style
- opportunity_time

原因：

- 这是输出契约的一部分

#### 用户建模逻辑

包括：

- 单会话画像生成
- 多会话聚合画像生成
- 身份标记提取
- 概率趋势分析

原因：

- 这是分析能力，不是存储能力

重要说明：

- 用户建模属于 Skill
- 长期持久化记忆不属于 Skill

#### 时间特征脚本

包括：

- 星期提取
- 深夜标记
- 周末标记
- 假期提示

原因：

- 这是确定性分析能力
- 可以稳定复用
- 非数据库型状态能力

重要说明：

- 脚本归 Skill
- 由 Skill 内部工作流按条件调用
- 如果输入里已有明确时间描述，Skill 可直接使用原始时间证据

#### 可选内置 RAG

包括：

- 本地检索脚本
- 固定索引或固定语料
- 检索结果格式标准化
- 从训练集导出的 retrieval assets

原因：

- 如果索引已稳定，它属于领域知识资产
- 适合一起打包交付

前提：

- Skill 仍然必须支持无 RAG 运行
- 内置 RAG 的调用决策由 Skill 内部工作流负责

重要区分：

- 进入 Skill 的应是“运行用 retrieval assets”
- 不应直接把 `benchmark/train.jsonl` 以研发数据集身份塞进 Skill
- `val/test/trigger eval` 更不能进入 Skill

### 4.2 不应属于 Skill 的内容

#### 长期画像数据库

包括：

- `user_id -> 历史画像` 持久化存储
- 查询某用户过去所有会话
- 自动更新长期画像快照

原因：

- 这是状态管理
- 涉及隐私和治理
- 与部署环境强绑定

#### 历史数据自动回填

包括：

- 每次调用自动去库里查历史
- 每次调用结束自动写回数据库

原因：

- 这是运行时编排职责
- 不适合作为可移植 Skill 的默认行为

#### 外置 RAG 服务运维

包括：

- 向量库部署
- embedding API 管理
- 检索服务 API
- 索引更新任务

原因：

- 属于系统服务能力，不属于 Skill 本体

#### 网站展示逻辑

包括：

- 卡片 UI
- 曲线图
- 页面导航
- 解释页文案

原因：

- 属于展示层

---

## 5. 外部运行时系统边界

### 5.1 必须属于运行时系统的内容

#### 输入标准化

包括：

- 读取聊天记录
- 读取 JSON/JSONL 文件
- 组织多会话 payload

#### 时间特征预填（可选高级集成）

说明：

- 外部系统只需调用 Skill 的统一入口
- 是否调用时间脚本由 Skill 内部工作流根据 `SKILL.md` 里的适用条件决定
- 这里不是第二套时间逻辑
- 如果输入里已经有像“2026-03-18 周三 8:59”这样的明确时间描述，则无需强制调用脚本
- 只有在某些宿主系统希望显式预填 `time_features` 时，才需要做额外预处理

#### 外置 RAG 提供层（可选高级集成）

包括：

- 对接外置检索服务
- 将检索结果标准化成 `retrieved_cases`

说明：

- 默认路径下，宿主系统不需要显式调内置 RAG
- 内置 RAG 由 Skill 内部工作流负责

#### 长期画像摘要构造

包括：

- 从数据库读取历史分析结果
- 压缩为 `prior_profile`
- 传入 Skill

#### 结果归档

包括：

- 保存每次分析结果
- 保存画像快照
- 保存趋势数据

### 5.2 运行时系统中的“长期记忆”定义

长期记忆在系统层的含义是：

- 不是简单把历史对话传给 Skill
- 而是长期保存用户过去的分析结果和画像状态
- 供未来查询、压缩、回填和产品化使用

系统层应支持两种历史使用方式：

#### 方式 A：全量历史传入 Skill

适用：

- 样本少
- demo 环境
- 研究模式

优点：

- 透明

缺点：

- payload 变大

#### 方式 B：历史摘要传入 Skill

适用：

- 正式系统
- 多次调用
- 数据库环境

优点：

- 高效
- 稳定

缺点：

- 需要额外摘要构造逻辑

## 6. 研发工具链边界

### 6.1 属于研发工具链的内容

#### evaluator

职责：

- 评测输出质量
- 计算准确率/F1
- 分析边界错误

不进入 Skill。

#### optimizer

职责：

- 根据错误样本优化 Skill 正文中的推理规则
- 后续应单独优化 `description`

不进入 Skill。

#### pipeline

职责：

- 组织完整研发流程
- 跑多轮演化
- 保存报告

不进入 Skill。

#### benchmark 数据集

职责：

- 训练/验证/测试
- trigger eval
- 多会话趋势 eval

不进入 Skill。

补充说明：

- benchmark 中的 `train` 可以作为内置 RAG 资产的来源
- 但正式进入 Skill 的应是从训练集导出的 retrieval assets，而不是 benchmark 原件
- `val/test/trigger eval` 始终只属于研发评测资产

---

## 7. 网站边界

网站是体验系统，不是 Skill 本体。

网站负责：

- 输入用户案例
- 调用运行时系统
- 展示 Skill 结果
- 可视化趋势轨迹
- 展示方法和数据来源

网站不负责：

- 改写 Skill 的输出字段
- 替代 Skill 的判定逻辑

## 9. 决策规则

后续如果再遇到一个组件不确定放哪里，按这四个问题判断：

### 问题 1

这个能力是不是“未成年人识别本身”的分析逻辑？

- 是：优先放 Skill
- 否：继续看问题 2

### 问题 2

这个能力是不是“状态持久化/查询/调度”？

- 是：放运行时系统
- 否：继续看问题 3

### 问题 3

这个能力是不是“评测/优化/演化”？

- 是：放研发工具链
- 否：继续看问题 4

### 问题 4

这个能力是不是“可视化展示/交互体验”？

- 是：放网站系统

---

## 10. 当前冻结结论

本项目当前正式冻结以下边界：

1. 用户建模属于 Skill 内部核心能力。
2. 长期画像持久化属于 Skill 外部运行时系统。
3. 时间脚本可以内置在 Skill 中，并由 Skill 内部在时间信息不完整时按条件调用。
4. RAG 可支持内置与外置两种来源，但 Skill 只消费标准化检索结果。
5. evaluator、optimizer、pipeline 全部属于研发工具链，不进入交付 Skill。
6. 网站是体验系统，不等于 Skill 本体。
